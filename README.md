# materialsFilter

> Программа для фильтрации списка соединений по заданным критериям (наборам химических элементов).

**Version:** 1.0.0  
**Language:** Go  

---

## 📌 Краткое описание

`materialsFilter` позволяет отфильтровать соединения из CSV-файла на основе пользовательских фильтров — наборов химических элементов. Результаты сохраняются в отдельные файлы в папке `output`.

---

## ✨ Возможности

- **Управление фильтрами**:
  - ✅ Добавление нового фильтра с уникальным именем и набором элементов  
  - ✅ Добавление элемента(ов) в существующий фильтр  
  - ✅ Удаление элемента(ов) из фильтра  
  - ✅ Удаление фильтра по имени  
  - ✅ Полная очистка всех фильтров  

- **Настройка данных**:
  - ✅ Указание и изменение имени CSV-файла с исходными данными
  - ✅ Указание и изменение имени CSV-файла с результатами обработки
  - ✅ Изменение заголовка таблицы с результатами фильтрации (после фильтрации)

- **Интерактивный интерфейс**:
  - ✅ Информационные сообщения в терминале  
  - ✅ Подсказки при вводе  
  - ✅ Проверка корректности ввода с повтором при ошибке  

---

## ⚠️ Ограничения

- Поддерживается только разделитель `;`
- Обрабатывается только первый столбец CSV-файла
- Не выполняется проверка корректности химических формул

---

## 🛠 Требования

- **ОС**: Windows / Linux  
- **Формат входных данных**:  
  - Файл в формате CSV 
  - Первый столбец — список соединений (например: `NaCl`, `H₂SO₄`, `CaCO₃`)  
  - Разделитель: только точка с запятой `;`  
  - Первая строка таблицы считается заголовком и игнорируется при обработке

## 📂 Структура проекта

При первом запуске создаются следующие каталоги:

```
materialsFilter/    ← директория, в которой запускаете программу
├── filter.exe (или `filter` — в зависимости от платформы)
├── data/
│   ├── input/      ← сюда поместите ваш CSV-файл
│   └── output/     ← сюда сохранятся результаты

```

> ⚠️ **Важно**: Прежде чем продолжить работу в программе, поместите файл с исходными данными в папку `data/input` и укажите его название.

## 🚀 Инструкция по запуску

1. **Запустите программу**:
- Windows: `filter.exe`
- Linux: `./filter`

2. **При первом запуске**:
  - Программа попросит вас поместить файл с исходными данными в папку `input`.
  - Введите **имя файла с исходными данными** (латиница, цифры, символы `_` и `-`, например: `compounds.csv`).
  - Программа сообщит, что конфигурация отсутствует.
   - Следуйте подсказкам в терминале:
     - Введите **имя фильтра** 
     - Введите **имя файла с результатами обработки**
     - Введите **элементы фильтра**

   > Если ввод некорректен — программа попросит повторить.
   > Ожидается подтверждение ввода после каждой операции.

3. **Настройка фильтров**:
   Доступные команды (вводятся в терминале):

   ```
   add-filter          — создать новый фильтр
   del-filter          — удалить фильтр
   clear-filters       — удалить все фильтры
   add-elements        — добавить элементы в фильтр
   del-elements        — удалить элементы из фильтра
   set-input-file      — изменить имя входного файла
   set-output-name     — изменить имя выходного файла
   show-config         — показать текущие настройки
   help                — показать список команд
   run                 — запустить фильтрацию
   ```

После ввода команды следуйте инструкциям в терминале.

4. **Изменение шапки таблицы**
После создания соответствующих фильтров введите команду `run`. В терминале появится шапка таблицы и одна строка с данными. Программа предложит изменить заголовки столбцов. Следуйте инструкциям в терминале. Наконец, файл(ы) с результатами фильтрации сохраняется(ются) в `data/output/`.

5. **Формат ввода:**
	- **Имя фильтра** - латиница, цифры, без пробелов, разрешены символы `_` и `-` (например: `heusler`)
	- **Имена файлов** - латиница, цифры, без пробелов, разрешены символы `_` и `-`, необходимо указать расширение `.csv` (например: `compounds.csv`)
	- **Символы химических элементов** - вводятся по одному в строке, регистр учитывается (например: `Cr`)
	- **Список химических соединений** - учитывается только регистр (например: `Co2FeAl`). 
	- **Команды** - вводится только имя команды
	- **Заголовки столбцов** - ограничений не имеется, ввод пустой строки предполагает использовать оригинальный заголовок.

---

## ⚙️ Принцип фильтрации

Соединение считается подходящим, если:
- оно содержит только элементы, указанные в фильтре
- регистр символов учитывается

Пример:
Фильтр: Co, Fe, Al  
Подойдут: Co2FeAl  
Не подойдут: CoAl

---

## 🔍 Извлечение символов химических элементов

Используется регулярное выражение `[A-Z][a-z]?`, которое находит:
- одну заглавную латинскую букву
- за которой может следовать одна строчная буква
Если в списке соединений встречается строка `Compaund`, регулярное выражение выделит символ `Co`,  и строка может быть ошибочно добавлена в результаты фильтрации.
